# Bus Seat Booking System 🚌

A lightweight, full-stack web application that allows passengers to view available bus routes, select seats using a visual 2x2 layout, and book tickets. It also includes an admin dashboard for managing bookings and updating seat statuses.

## 🚀 Features

**Passenger Interface:**
* View available bus routes and travel schedules.
* Interactive 2x2 visual seat selection map.
* Real-time seat availability (booked seats are disabled and highlighted).
* Instant booking confirmation with unique reference IDs.

**Admin Interface:**
* View all passenger bookings in a data table.
* Cancel or update passenger bookings dynamically.
* Seat availability instantly updates on the frontend upon cancellation.

## 🛠️ Tech Stack

* **Backend:** Python, FastAPI, SQLAlchemy
* **Database:** MySQL
* **Frontend:** HTML5, CSS3, Vanilla JavaScript
* **Architecture:** RESTful API with a decoupled frontend/backend structure

## ⚙️ Local Setup & Installation

### 1. Clone the Repository
```bash
git clone https://github.com/Induwara24/bus-booking-system.git
cd bus-booking-system
```

### 2. Database Setup
1. Ensure you have a local MySQL server running.
2. Create a new database named `bus_booking_db`:
   \`\`\`sql
   CREATE DATABASE bus_booking_db;
   \`\`\`
3. Update the `SQLALCHEMY_DATABASE_URL` in `backend/database.py` with your MySQL credentials.

### 3. Backend Setup
1. Navigate to the backend directory:
   \`\`\`bash
   cd backend
   \`\`\`
2. Create and activate a virtual environment:
   \`\`\`bash
   python -m venv venv
   venv\Scripts\activate   # On Windows
   source venv/bin/activate  # On macOS/Linux
   \`\`\`
3. Install the dependencies:
   \`\`\`bash
   pip install -r requirements.txt
   \`\`\`
4. Run the FastAPI server:
   \`\`\`bash
   uvicorn main:app --reload
   \`\`\`
   *The server will start at `http://127.0.0.1:8000` and automatically create the necessary database tables.*

### 4. Frontend Setup
Simply open `frontend/index.html` or `frontend/admin.html` in any modern web browser. No frontend build tools or local servers are required!

## 📡 API Endpoints Summary

* `GET /routes` - Fetch all available bus routes.
* `GET /routes/{route_id}/buses` - Fetch schedules for a specific route.
* `GET /buses/{bus_id}/seats` - Fetch an array of currently booked seat numbers.
* `POST /bookings` - Create a new passenger booking.
* `GET /admin/bookings` - Fetch all bookings for the admin dashboard.
* `PUT /admin/bookings/{booking_id}/status` - Update a booking's status (e.g., Cancelled).

Interactive API documentation is automatically generated by FastAPI and available at `http://127.0.0.1:8000/docs` when the server is running.
